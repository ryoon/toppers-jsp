１．RISC-V ターゲット依存部の概要


1.1 ターゲットシステムと開発環境

RISC-V プロセッサのターゲットシステムとしては，RISC-VアーキテクチャのCPU
コアを搭載した以下のボードをサポートしている．

  ・Raspberry Pi Ltd.製 Rapberry Pi Pico2(RISC-V32:hazard3)
  ・Efinitx社T120 BGA324ボード(Cortex-V:Sapphire SoC)

開発環境には，GCC などの GNU開発環境を用い，オブジェクトファイルフォー
マットは ELF を標準とする．


1.2 サポートする機能の概要

RISC-V 依存の機能として、性能評価用システム時刻参照機能(vxget_tim)をサポ
ートしている．割込みの禁止と許可(dis_int、ena_int)はサポートしている．


1.3 他のターゲットへのポーティング

他のターゲットにポーティングするためには，システム依存部に以下のコード
が必要となる．
 
  ・スタートアップ・ルーチン
  ・シリアルドライバ(hw_serial.c)


1.4 スタートアップ・ルーチン

スタートアップ・ルーチンの記載はROM実行とRAM実行で異なる、
スタートアップ・ルーチン中でコンパイル・スイッチ
TOPPERS_RAM_EXECにて、処理を切り分けている．
各ボードのスタートアップ・ルーチンは以下の通り．
T120 BGA324はRAM実行のみ
  ・Rapberry Pi Pico2：rp2350.c
  ・T120 BGA324：なし、FPGAのためスタートアップ・ルーチンは不要

1.5 シリアルポート

Rapberry Pi Pico2 ではUART0ポート をカーネルログ出力用に使用する．
ディフォルトでは，速度115200bps,データ:8bit,Parity:none,Stop:1bitであ
る．
T120 BGA324 ではSYSTEM_PLIC_SYSTEM_UART_0ポート をカーネルログ出力用
に使用する．ディフォルトでは，速度115200bps,データ:8bit,Parity:none,
Stop:1bitである．


２．RISC-V プロセッサ依存部の機能

この節では、カーネルおよびシステムサービスの機能の中で、
RISC-V依存の部分について解説する．


2.1 データ型

int型および unsigned int型のサイズは 32ビットである．
コアにより、64ビット対応のものもある、
この場合、longまたはunsigned longが基本型となる．

2.2 割込み管理機能と割込みハンドラ

RISC-Vの割込み許可禁止はmstatusのMSTATUS_MIEとmieの割込みビットを併用
して使用する．但し、基本はmie制御となる．
mieをどのように制御するかuser_config.hに記載している．
DEF_INH で指定する割込みハンドラ番号(inhno)は、各CPUの割込みベクタ設定
に従う．標準の割込み機構はPLICとなるが、コア毎に制御やインターフェースが
異なるため、インライン関数でシステム部で記載している
割込み優先度は大きい値が優先度が高い、レベルもコアにより差異がある．
データ型(INHNO)はunsigned int型に定義されている．

2.3 CPU例外管理機能とCPU例外ハンドラ

DEF_EXC で指定する割込みハンドラ番号(excno)は、RISC-Vで16個指定され、
コア毎に有効な番号が定義されている，
例外要因である．そのデータ型(EXCNO)は unsigned int 型に定義されている．
DEF_EXC で、例外コード番号として有効でない値や、CPU例外に対応しない番号
を指定した場合の動作は保証されない．

CPU例外ハンドラに渡されるp_excinfは，CPU例外発生時のコンテキストを保存
したスタックへのポインタが渡される．スタックの構造を以下に示す．
以下は、基本の設定であり、コア毎にアレンジされる場合もある．


      ----------------------
     |       mstatus        |  <-- p_excinf
      ----------------------
     |        mpc           |
      ----------------------
     |       mcause         |
      ----------------------
     |        mip           |
      ----------------------
     |       sp(x2)         |
      ----------------------
     |      reserved        |
      ----------------------
     |       a0(x11)        |
      ----------------------
     |       a1(x11)        |
      ----------------------
     |       ra(x1)         |
      ----------------------
     |       a2(x12)        |
      ----------------------


2.4 スタートアップモジュール

RISC-V 依存のスタートアップモジュール(start.S)では、次の初期化処理を行う．

(A) プロセッサモードの初期化とスタックポインタの初期化

最初に、グローバルポインタの設定をおこない
次に、制御レジスタの初期化これにより割込みを禁止にする．
スタックポインタ(sp)を STACKTOP に設定する．ここで割込みスタックポインタに
設定されたスタック領域は、カーネル起動後は非タスクコンテキスト用のスタック
領域として使われる．STACKTOP は、sys_defs.h 部で定義することを想定している．
次に、実行コアの確認を行いコア０以外は(B)の初期は行わない．


(B) bssセクションと dataセクションの初期化

bssセクションをゼロクリアする．また、dataセクションを初期化する．

(C) FPUの初期化

コンパイル・スイッチ__riscv_float_abi_softが定義されていなければ
FPU初期化する．

(D) カーネルの起動

kernel_start へ分岐し、カーネルを起動する．kernel_start からリターンし
てくることは想定していない．


３．システム依存部の機能


3.1 システムクロックドライバ

システムクロックドライバが isig_tim を呼び出す周期は、sys_defs.h 中の 
TIC_NUME と TIC_DENO で定義されている(標準は 1ミリ秒周期)．この定義
を変更することで、isig_tim を呼び出す周期を変更することができる．ただ
し、タイマの精度が 1μ秒であるため、1μ秒単位で端数になる値を設定した
場合には、isig_tim の呼出し周期に誤差が生じることになる．


3.2 性能評価用システム時刻参照機能

RISC-V 依存部では、性能評価用システム時刻参照機能(vxget_tim)をサポートし
ている．性能評価用システム時刻の精度は 1μ秒単位であるが、タイマの現在値
を読み出すために一時的にタイマを停止させる必要があるため、vxget_tim を呼
ぶ度にシステムクロックが少しづつ遅れることになる．なお、SYSUTIM型は UD型
(64ビットの符号無し整数型)に定義している．


3.3 シリアルインタフェースドライバ

各システム依存部では，hw_serial.c/h/cfgにてUART をサポートしている．


４．開発環境の構築

開発環境の構築方法については，GNU開発環境構築マニュアルを参照のこと．


4.1 開発環境のバージョン
動作確認したツールのバージョンは以下の通りである．

  gcc version 8.2.0(riscv-none-embed)：windows
  gcc version 12.2.0(riscv64-unknown-elf)：ubuntu

4.2 ツール・ソースに関して

4.2.1 フラッシュROMへの書き込み
  Pico2のプログラムをフラッシュROMに書き込む場合、RP2350用のuf2ファイルに
　変換を行う必要がある．
　binファイルをuf2ファイルに変換するプログラムbin2uf2をtool/bin2uf2に用意
　している．このソースファイルをセルフのgccでビルドしてusr/local/binに
　置けば、コマンドとして使用することができる．
　R2350形式のフォーマット設定がオプション -family 1 を指定すれば良い．

  Sapphire SoCはROM実行に対応していない．
  作者の環境では、UART0のピンをFPGAのUART設定は、
　TXがGPIOT_RXP20(B16)でJ12コネクタのPIN1、RXがGPIOT_RXP21(B15)で
　J12コネクタのPIN2に配線している．
　また、ビルド時ハードウェア情報を取り込むためにFPGAの生成時作成される
　以下のファイルをシステム部に置いて、ビルドを行う必要がある．
　　・soc.mk
　　・soc.h

4.2.2  ROMモニタに関して
　Pico2ではuf2形式のRISC-V用ROMモニタをtoolsディレクトリに置いている．
    ・rommon_pico2rv.uf2

  Sapphire SoCの実行はFPGAに小型のROMモニタを書き込み、これを使って
　SRAMにダウンロード実行を行う形式のみをサポートする．
  小型のROMモニタはtools/t120f324_monitor
  これをembedded_sw/SapphireSoC/software/standaloneに置いてFPAGAを作成
　すれば、小型のROMモニタをもつFPGAが作れる．

4.2.3  不具合に関して
　c++言語でthow/catchを使用するとaboutが発生する．
　cpp.txtを参照のこと．

4.2.4 GDB
   対応を行っていない．
