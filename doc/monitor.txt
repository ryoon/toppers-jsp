このドキュメントでは，JSPカーネルのタスクモニタについて解説する．
タスクモニタはタスクレベルで動作し、他のタスクやタスクリソースの管理
や監視を行う．
特徴として、デフォルトのコマンドに加えsetup_command関数を使って、
コマンドの拡張を行うことができる．
コマンドは[カテゴリ]と[サブコマンド]で構成され、カテゴリ単位で
コマンドの拡張が行える．


(1) タスクモニタのインストール

Makefileとコンフギュレーション・ファイル(.cfg)にタスクモニタの
取り込み文を追加するだけでタスクモニタをインストールすることが
できます．

(1-1) Makefileの記載

Makefile中に、以下のインクルードを追加することにより、タスクモニタの
ソースを取り込むことができます．

#
#  ミドルウェアの Makefile のインクルード
#
include $(SRCDIR)/monitor/Makefile.config

(1-2) コンフギュレーション・ファイルの記載

コンフギュレーション・ファイルにmonitor.cfgを追加する．

#include "../../../monitor/monitor.cfg"

タスクモニタのsample1プログラムを作成するのは、
以下のコンフィギュレーションスクリプトを使って作成ができます．

    % mkdir OBJ
    % cd OBJ
    % perl ../configure -C armv7m -S nucleo_f401re -d monitor


(1-3) 実行状態

追記後、ビルドを行うとMONTASKがタスクとしてタスクモニタの機能を
処理します．タスク優先度はMONITOR_PRIORITY(4)となります．
タスクモニタはデフォルトとしてシリアルデバイスと通信を行い
コマンドの入力やレスポンスの表示を行います．
通信するシリアルデバイスはMONITOR_PORTIDに設定されますが、
この設定はデフォルトではJSPカーネルで指定するCONSOLE_PORTID
となります．
CDEFSの設定で、MONITOR_PORTIDを独自に設定すれば、シリアルデバイスを
変更でいます．

タスクモニタの入出力機能は、_SMALL_STDIO_設定で動作する独自の
標準入出力機能となります．
monitorディレクトリ中のstdio.hをインクルードすれば、アプリケーションで
C言語の標準入出力関数を利用することができます．

(2) 組込み用標準入出力

タスクモニタは組込み用の標準入出力機能を用いて、コマンド入力、表示を
行っている．この機能は、monitorディレクトリ中のstdio.hを参照することで
有効となる．標準入出力機能はライブラリィの機能を使う方法もあるが、
簡単なプログラムで記載することで、ユーザーにも達成方式を理解してもらう
ことを第一として、ソースで記載している．
iobで定義する３つの標準ファイル構造体(FILE)で標準入出力を定義し、
iboのファイルを切り替えることで、他のストリームと入れ替えが可能な
構造としている．

(3) コマンド実行

コマンドはプロンプト(mon>)表示状態で、コマンドラインを入力し、リターン
キーで実行します．コマンドラインは、以下のように

[カテゴリ] [サブコマンド] [パラメータ１]　[パラメータ２] … [パラメータN]

の順に入力を行います．

(3-1) デフォルト・コマンド

  Display   BYTE    [start address] [end address]
            HALF    [start address] [end address]
            WORD    [start address] [end address]
                    アドレス [startaddr] から、[end address]までの
　　　　　　　　　　メモリの内容をBYTE(1バイト)、HALF(2バイト)
　　　　　　　　　　WORD(4バイト)単位でデータを出力します．
　　　　　　　　　　[startaddr]を省略すると、前回出力したデータの
　　　　　　　　　　次から 128 バイト出力します．
　　　　　　　　　　アクセスできないアドレスを指定した場合はオールビット
　　　　　　　　　　オンの値を出力します．
            REGISTER
                    表示可能な場合は、停止中のタスクのレジスタ値を表示
                    します．
  Set       BYTE    [start address]
            HALF    [start address]
            WORD    [start address]
                    アドレス [startaddr]のメモリの内容をBYTE(1バイト)、
                    HALF(2バイト)、WORD(4バイト)単位でデータと"="を
                    表示します．
                    HEX数字を入力し、リターン・キーを押すとメモリの内容を
                    入力した数字に置き換えます．
                    リターン・キーのみ押すと、内容を変更せず、次のメモリの
                    データと"="を表示します．
                    .とリターン・キーを押すと入力操作をやめプロンプトに
                    戻ります．
            COMMAND [mode]
                    デフォルトmode == 2で、入力したカテゴリとサブコマン
                    ドを比較して判定する．
                    mode == 1で、入力したカテゴリとサブコマンドの最初の
                    １文字を比較して判定する．
            SERIAL  [port id]
                    タスクモニタ入出力のシリアルデバイスをport idに
                    変更します．
            TASK    [task id]
                    Taskコマンドの対象のタスクをtask idにタスクに変更し
                    ます．

  Task      ACTIVATE
                    対象のタスクにact_tskサービス・コールを発行する．
            TERMINATE
                    対象のタスクにter_tskサービス・コールを発行する．
            SUSPEND
                    対象のタスクにsus_tskサービス・コールを発行する．
            RESUME
                    対象のタスクにrsm_tskサービス・コールを発行する．
            RELEASE
                    対象のタスクにrel_tskサービス・コールを発行する．
            WAKEUP
                    対象のタスクにwup_tskサービス・コールを発行する．
            PRIORITY [pri]
                    対象のタスクの優先度をpriに変更する．

  Log       MODE    [logmask] [lowmask]
                    システムログの出力マスクをlogmaskとlowmaskに変更し
                    ます．
  Suspend           [time]
                    タスクモニタをtime秒サスペンドする．
                    他のタスクがシリアルデバイスから入力を行っている
                    場合、そのタスクがシリアル入力できるようになる．
  Help
                    ヘルプ・メッセージを表示する．

(3-2) デフォルト・コマンド拡張

デフォルト・コマンドコンパイルスイッチの指定により、拡張コマンドを
使用できる．

(3-2-1) タスク実行時間の測定

ビルド時、LOG_ANALYSISコンパイルスイッチを定義すると
タスク実行時間の測定機能が追加される．

  Log       TASK    [cycle time(ms)] [count]
                    サイクル時間内のタスクの実行時間をcount回数表示する．

(3-2-2) デバイスアクセス表示

ビルド時、LOG_ANALYSISコンパイルスイッチを定義すると
タスク実行時間の測定機能が追加される．
但し、この機能を有効化するにはデバイスへのデータ書き込み時、ana_wdv関数
デバイスへのデータ読み出し時、ana_rdv関数を呼び出すようにデバイスドライバ
を書き換える必要がある．

  Log       PORT    [no] [logmask] [portaddress]
                    ログ対象のポートのポートアドレスとLOGMASKを設定する．
                    ana_wdv及びana_rdv関数呼び出し、設定のアドレスにマッチ
                    すれば、ログ表示を行う．


(4) 新規に作成したカーネルのタスクモニタの対応

タスクモニタの対応は任意なので、興味のない方はこの対応は不要です．
新規にconfigディレクトリにカーネルを追加した場合、
ターゲット依存に以下の３つの対応が必要です．

(4-1) タスクモニタのターゲット依存部の作成

monitor/archにターゲット依存部のソースを置いています．
例えば、ARMv7mの依存用ソースはarmv7m.cで、
プロセッサまたはシステム依存のディレクトリ中のMakefile.config中の
開発環境名の定義の定義の MARCHの指定が、ファイル名となります．
ターゲット依存となるのが、以下の２つです．

(4-1-1) T_MEMDEF const memdefine

  ターゲットボードのメモリマッピングを定義するデータ．

(4-1-2) display_registers(ID tskid)

  待ち状態のレジスタ表示を行う関数、レジスタの表示は
　割込みで待ち状態となった場合のみで、スケジューラ中のret_int_rが
　停止位置となる場合のみです．このラベル位置に_kernel_break_wait
　にグローバル宣言をする必要があります．
　（このグローバル宣言が無くても、リンクエラーとならない）

(4-1-3) タスクの実行時間の設定

　LOG_ANALYSISコンパイルスイッチ宣言で、タスクの実行時間を測定できます
　が、この機能を有効にするために、dispatcherを呼び出した時点で
　iana_tsk関数を呼び出すように修正が必要です．
　以下に、ARMv7m中のcpu_support.S中の対応の場所のプログラムを示します．

/*
 *  ディスパッチャ本体
 */
dispatcher:
	/*
	 *  このルーチンは，タスクコンテキスト・CPUロック状態・ディスパッチ
	 *  許可状態・（モデル上の）割込み優先度マスク全解除状態で呼び出さ
	 *  れる．
	 *
	 *  すなわち，Threadモード・enadspがtrue
	 *  となっている．実行再開番地へもこの状態のまま分岐する．
	 */
#ifdef CONTROL_FPU_REGISTERS
	mrs r3, control			/* ディスパッチャ実行時はFPCAを一律クリアする */
	bic r3, r3, #CONTROL_FPCA
	msr control, r3
#endif /* CONTROL_FPU_REGISTERS */
	ldr r1, =schedtsk		/* schedtskをruntskに */
	ldr r0, [r1]
	ldr r2, =runtsk			/* schedtskをruntskにセット */
	str r0, [r2]			/* (タスクスイッチの実行) */
#ifdef	LOG_ANALYSIS
	push {r0-r3}
	bl iana_tsk				/* タスク実行情報の設定 */
	pop {r0-r3}
#endif	/* 	LOG_ANALYSIS */
	cbz r0, dispatcher_1	/* runtskがNULLならdispatcher_1へ */
	ldr sp, [r0,#TCB_sp]	/* タスクスタックを復帰 */
	ldr pc, [r0,#TCB_pc]	/* 実行再開番地を復帰   */

(5) その他

(5-1) ディレクトリ・ファイル構成

タスクモニタの各ファイルの概要は次の通り．

    minitor/
        Makefile.config Makefileのインクルード
        monitor.cfg     タスクモニタのコンフィギュレーションファイル
        monitor.c       タスクモニタのソース
        monitor.h       タスクモニタの構成定義
        stdio.h         組込み標準入出力のインクルードファイル
        Makefile        SAMPLE1のタスクモニタ版Makefile
        task_expansion.c
                        JSPカーネル拡張関数
        task_expansion.h
                        JSPカーネル拡張定義
        sample1.h       SAMPLE1のタスクモニタ版定義
        sample1.cfg     SAMPLE1のタスクモニタ版
                        コンフィギュレーションファイル
        sample1.c       SAMPLE1のタスクモニタ版関数
        sample1.h       SAMPLE1のタスクモニタ版定義

    monitor/arch
        armv6m.c        タスクモニタARMv6m依存部
        armv7m.c        タスクモニタARMv7m依存部
        armv8m.c        タスクモニタARMv8m依存部
        riscv32.c       タスクモニタRISC-V32依存部

    monitor/stdio
        stddev.c        組込みSTDIO関数
        printf.c        printf関数
        fprintf.c       fprintf関数
        sprintf.c       sprintf関数
        vsnprintf.c     vsnprintf関数
        scanf.c         scanf関数
        sscanf.c        sscanf関数
